# Even Crypto HUD

A small TypeScript Even Hub app for showing a crypto watchlist on Even G2 glasses. The MVP uses CoinGecko as the read-only price provider, lets the user enter their own API key, and renders the first four watchlist coins in a pixel-style card HUD.

This app does not trade, place orders, or connect to wallets.

## MVP Scope

- Even G2 glasses HUD rendered through `@evenrealities/even_hub_sdk`
- CoinGecko Demo API key onboarding in the phone WebView
- CoinGecko coin catalog search by symbol, name, or id
- Watchlist chip controls for add, remove, move up, and move down
- Watchlist sync through Even App bridge storage, plus a manual Refresh watchlist button
- First four watchlist coins rendered on glasses
- `LAST UPDATED` plus local time shown at the top-right of the HUD, aligned with the first row
- No 24h comparison text in the glasses layout

## Requirements

- Node.js 20+
- npm
- Even Realities App with Even Hub / Developer Center access
- Even G2 paired to the phone for hardware testing
- A CoinGecko Demo API key

Useful references:

- Even Hub CLI docs: https://hub.evenrealities.com/docs/reference/cli
- Even first app walkthrough: https://hub.evenrealities.com/docs/getting-started/first-app

## Run Locally

```bash
npm run dev
```

The dev script starts Vite with `--host 0.0.0.0`, so the app can be opened from another device on the same Wi-Fi network.

Typical local URL:

```text
http://localhost:5173/
```

Typical phone URL:

```text
http://<your-mac-lan-ip>:5173/
```

Use the LAN IP that Vite prints in the terminal, for example `http://192.168.1.23:5173/`. Do not use `localhost` or `127.0.0.1` on the phone; those point to the phone itself.

## Push To Phone For Testing

For MVP testing, use Even Hub QR sideloading. This loads the running dev server on the phone, so you get a fast edit-refresh loop while the server stays running on your Mac.

1. Put the Mac and phone on the same Wi-Fi network.
2. Start the dev server:

   ```bash
   npm run dev
   ```

3. Generate a QR code for the dev server URL:

   ```bash
   npx evenhub qr --url http://<your-mac-lan-ip>:5173/
   ```

   Example:

   ```bash
   npx evenhub qr --url http://192.168.1.23:5173/
   ```

   You can also let the CLI remember the IP/port interactively:

   ```bash
   npx evenhub qr
   ```

4. Open the Even Realities App on the phone.
5. Go to Even Hub / Developer Center.
6. Use the built-in scanner to scan the QR code from the terminal.
7. Open the app from the phone and enter the CoinGecko Demo API key.
8. Confirm the glasses show the watchlist card.

### Finding Developer / Prototype Mode

The exact phone-app labels can vary by Even Realities App version and account state. The official docs refer to a "Developer Center" inside the Even Realities App. Some current community reports describe the QR sideload entry as "Prototype mode".

Try this path first:

1. Open the Even Realities App.
2. Tap the Even Hub tab.
3. Tap the glasses icon in the top-right area.
4. Tap the account/profile row that shows your username or email.
5. Look for "Developer Center" or "Prototype mode".
6. Open it and scan the QR code generated by `npx evenhub qr`.

If "Developer Center" or "Prototype mode" is still missing, the account may not be enabled for developer testing yet, or the phone app/glasses firmware may need updating.

### If Even Hub Or Developer Center Is Missing

The QR and private-build flows both depend on Even Hub support in the phone app. According to the official support docs, Even Hub is designed for Even G2 and should appear as an "Even Hub" tab in the bottom navigation after updating the Even Realities App to the latest version.

If the tab is not visible:

- Update the Even Realities App from the App Store or Google Play.
- Confirm the phone is paired with an Even G2 device.
- Confirm the glasses firmware is up to date.
- Sign in with the same Even Realities account you use for the Even Hub developer portal.
- Force quit and reopen the phone app after updating or pairing.
- Check the profile / plugin management area for "My plugins", "Beta", or "Developer Center" entries.
- If the app still has no Even Hub entry, the current phone app/account may not have Even Hub enabled yet; continue with simulator testing and contact Even support or the developer program before testing on hardware.

If scanning fails:

- Make sure the phone can open `http://<your-mac-lan-ip>:5173/` in its browser.
- Keep the Vite dev server running.
- Check macOS firewall/VPN settings.
- Make sure both devices are on the same network and not isolated by guest Wi-Fi.
- Regenerate the QR code with `npx evenhub qr --clear`.

## Install A Packed `.ehpk`

There are two different device-testing paths:

- QR sideloading: fastest for local development. It points the Even Realities App at your running dev server and does not use `.ehpk`.
- Private build / submission: used when you already have an `.ehpk`. Upload the package through the Even Hub developer portal, then install/test that build from the Even Realities App.

Current packed app:

```text
even-crypto.ehpk
```

Private build flow:

1. Verify and package the app:

   ```bash
   npm test
   npm run typecheck
   npm run build
   npx evenhub pack app.json dist -o even-crypto.ehpk -c
   ```

2. Open the Even Hub developer portal:

   ```text
   https://hub.evenrealities.com/
   ```

3. Sign in with the same Even Realities account used on the phone.
4. Create or open the app entry that matches `app.json`.
5. Upload `even-crypto.ehpk` as a private build or submission build.
6. Open the Even Realities App on the phone.
7. Go to Even Hub / Developer Center.
8. Find the uploaded private/test build and install it.
9. Launch it from the phone or glasses menu and run the MVP checklist below.

Notes:

- The phone app does not currently use the local `.ehpk` file directly for the normal dev loop.
- If you only want to test immediately on your own phone, use QR sideloading.
- If you want to test the packaged runtime, share a build, or prepare for review, use the developer portal upload path.

### If The Uploaded Build Does Not Refresh On Phone

When uploading a new `.ehpk`, bump the `version` in `app.json` before packing. The phone app may keep showing the previous build if the uploaded package still uses the same version.

Recommended release loop:

1. Bump `app.json.version`, for example from `0.1.0` to `0.1.1`.
2. Keep `package.json` and `package-lock.json` in sync with the same version.
3. Rebuild and pack:

   ```bash
   npm run build
   npm run pack
   ```

   If you are logged into the EvenHub CLI, you can additionally run `npx evenhub pack app.json dist -o even-crypto.ehpk -c` to check package availability.

4. Upload the new `even-crypto.ehpk` in the developer portal.
5. In the portal, make the newly uploaded version the active beta/private test build if that option is available.
6. Force quit the Even Realities App on the phone.
7. Reopen the phone app and check Developer Hub / My plugins again.
8. If the old build still appears, remove the old build from My plugins, force quit, reopen, and check again.

## Project Structure

```text

  even/
    bridge.ts                    Even App bridge detection/connection
    cryptoPage.ts                Glasses HUD container layout and text updates
  formatters/
    priceFormatter.ts            Price and HUD text formatting
  prices/
    coinCatalogSource.ts         CoinGecko /coins/list catalog fetcher
    coingeckoPriceSource.ts      CoinGecko /simple/price price fetcher
    mockPriceSource.ts           Mock provider for local app states/tests
  settings/
    apiKeyStore.ts               Browser localStorage API key store
    coinCatalogStore.ts          Cached CoinGecko catalog store
    watchlistStore.ts            Watchlist storage, migration, search, and chip helpers
  main.ts                        Browser UI, events, refresh loop, and glasses sync
  styles.css                     Phone WebView and preview styling
```

## Runtime Notes

- API key storage is synced to Even App bridge storage when the app runs inside Even Realities, with browser `localStorage` used as a local cache.
- Watchlist storage is synced to Even App bridge storage when the app runs inside Even Realities, with browser `localStorage` used as a local cache.
- CoinGecko catalog cache is kept for 30 minutes.
- Price refresh interval is 5 minutes.
- Only the first four watchlist coins are sent to glasses.
- `app.json` grants network access only to `https://api.coingecko.com`.

## Debug Logs

Simulator logs are the easiest way to inspect startup, storage sync, failed fetches, and uncaught errors:

```bash
npm run dev
npx evenhub-simulator http://localhost:5173/ --automation-port 9898
curl http://127.0.0.1:9898/api/console
```

Incremental polling:

```bash
curl "http://127.0.0.1:9898/api/console?since_id=0"
```

Glasses screenshot:

```bash
curl -fsS http://127.0.0.1:9898/api/screenshot/glasses -o /tmp/even-crypto-glasses.png
```

The app emits non-secret diagnostic logs with the `[even-crypto]` prefix. API key values are never printed.
